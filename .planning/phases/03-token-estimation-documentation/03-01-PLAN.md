# Plan 01: Token estimation function in ce-shared

## Objective
Add `estimate_tokens_from_cost()` to `ce-shared/src/ce_shared/pricing.py` — the inverse of `cost_for_model()`. Given a model name and USD cost, back-calculate estimated input and output token counts using a configurable input:output ratio (default 5:1). This fills the gap where SDK Agent calls report cost but not token counts.

## must_haves
- `estimate_tokens_from_cost(model, cost_usd, input_output_ratio=5.0)` exists in `ce_shared.pricing`
- Returns a dict with keys: `input_tokens` (int), `output_tokens` (int), `token_source` (str = `"estimated_from_cost"`)
- Zero cost returns `{"input_tokens": 0, "output_tokens": 0, "token_source": "estimated_from_cost"}`
- Unknown model defaults to Opus-tier pricing `(5.00, 25.00)` for estimation (TOKN-04)
- All returned token counts are non-negative integers; non-zero cost produces at least 1 token per field
- Round-trip accuracy: `cost_for_model(model, est_input, est_output)` is within 5% of original cost (rounding error)
- Function is re-exported from `ce_shared.__init__`
- Unit tests cover: normal estimation, zero cost, unknown model, round-trip accuracy, minimum-1 guarantee
