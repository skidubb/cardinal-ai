# Plan 03: Wire token estimation into production agent path and Langfuse

## Objective
Fix the production SDK agent path in `llm.py:agent_complete()` that currently bypasses `_record_usage()` entirely. After `agent.chat()`, capture `agent.cost`, call `estimate_tokens_from_cost()`, and feed the estimated tokens into both the cost tracker and Langfuse tracing with `token_source: "estimated_from_cost"` metadata.

## must_haves
- `agent_complete()` in `llm.py` no longer returns immediately after `agent.chat()` — it captures `agent.cost` and records usage
- Estimated tokens flow to `ProtocolCostTracker.track()` for the production agent path
- Estimated tokens flow to `langfuse_tracing.record_generation()` for the production agent path
- `token_source: "estimated_from_cost"` is included in Langfuse span metadata
- `record_generation()` accepts and passes through a `token_source` metadata field
- Real token counts from the research/fallback path are NOT overwritten by estimation
- Zero cost from SDK agent skips estimation (logs warning, does not produce fake tokens)
