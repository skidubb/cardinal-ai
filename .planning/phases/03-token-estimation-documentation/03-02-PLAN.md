# Plan 02: Budget guardrails in ProtocolCostTracker

## Objective
Add a warn-only cost ceiling to `ProtocolCostTracker` so that protocol runs exceeding a configured USD threshold log a warning but continue running. The ceiling is configurable via `PROTOCOL_COST_CEILING` env var with a sensible default.

## must_haves
- `ProtocolCostTracker.__init__()` accepts `cost_ceiling_usd: float | None` parameter
- When `cost_ceiling_usd` is set and `total_cost` exceeds it after a `track()` call, a warning is logged via Python `logging.warning()`
- Warning message includes: current total cost, ceiling value, protocol name (if available)
- Protocol run is NOT halted — warn only (per user decision)
- No warning is logged when cost is under the ceiling
- `PROTOCOL_COST_CEILING` env var is read and used as default ceiling when no explicit value passed
- `.env.example` updated with `PROTOCOL_COST_CEILING` entry and comment
- Unit tests cover: warning fires on breach, no warning under ceiling, env var loading
