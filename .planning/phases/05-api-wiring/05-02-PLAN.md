<objective>
Add client-disconnect cancellation (stops burning API credits when browser closes), fix context var isolation for concurrent runs, and add curated pipeline presets to the pipelines endpoint.

Purpose: Without disconnect cancellation, every closed browser tab leaves an orchestrator task running until completion, burning API credits with no consumer. Context var isolation prevents a subtle concurrency bug. Pipeline presets enable the UI to offer packaged workflows without manual configuration.

Output: Active task registry in runner.py, disconnect watcher in endpoint handlers, CancelledError handling, pipeline presets module, expanded tests.
</objective>

5. Add new `except asyncio.CancelledError` block BEFORE the `except Exception` block:
   ```python
   except asyncio.CancelledError:
       _active_run_tasks.pop(run_id, None)
       with Session(engine) as session:
           run = session.get(Run, run_id)
           if run:
               run.status = "cancelled"
               run.completed_at = datetime.now(timezone.utc)
               session.add(run)
               session.commit()
       yield _sse_event("run_complete", {"run_id": run_id, "status": "cancelled"})
   ```

**B. Context var isolation (API-09):**
1. The context var setup (set_cost_tracker, set_event_queue, set_no_tools) already happens before asyncio.create_task, which is correct since the task copies the context snapshot at creation time.
2. The real fix: consolidate the scattered `set_cost_tracker(None)` calls (currently at end of success path AND in except block) into a single `finally` block. Wrap the entire try/except in run_protocol_stream with try/finally that calls `set_cost_tracker(None)` and `set_event_queue(None)`. Remove the individual `set_cost_tracker(None)` calls from both the success and error paths.
3. Same treatment for `run_pipeline_stream`: move `set_cost_tracker(None)` and `set_session_id(None)` to a finally block.

**C. Disconnect watcher in protocols.py (API-10):**
1. Import `_active_run_tasks` from `api.runner`.
2. Wrap the SSE generator in `start_protocol_run` with a disconnect guard:
   ```python
   async def _guarded_stream():
       disconnect_watcher = asyncio.create_task(_watch_disconnect(request, run_id))
       try:
           async for chunk in run_protocol_stream(...):
               yield chunk
       finally:
           disconnect_watcher.cancel()

   async def _watch_disconnect(request: Request, run_id: int) -> None:
       while not await request.is_disconnected():
           await asyncio.sleep(0.5)
       task = _active_run_tasks.get(run_id)
       if task and not task.done():
           task.cancel()
   ```
3. The EventSourceResponse wraps `_guarded_stream()` instead of the raw generator.
