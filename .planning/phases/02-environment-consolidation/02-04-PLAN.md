# Plan 04: Build env_check diagnostic CLI

## Objective
Create the `python -m ce_shared.env_check` diagnostic tool that reports environment health across the monorepo: which `.env` was loaded, all key statuses grouped by project with redacted values, and warnings for stale per-project `.env` files.

## must_haves
- `python -m ce_shared.env_check` runs and produces a Rich-formatted report
- Report shows which `.env` file was loaded (full path) at the top
- Keys are grouped by project (Agent Builder, Orchestration, Evals, Docker/DB)
- Each key shows: name, redacted value (first 4 + last 4 chars with `***` middle), required/optional status, set/missing status
- Color-coded: green for set, red for missing required, yellow for missing optional
- Warns if stale per-project `.env` files are found (checks `CE - Agent Builder/.env`, `CE - Multi-Agent Orchestration/.env`, `CE - Evals/.env`)
- Unit tests cover: redaction logic, grouping logic, stale file detection

<task id="02-04-01">
Create `ce-shared/src/ce_shared/env_check.py` with:

1. `redact(value: str) -> str`: If len < 8, return `***`; else return `value[:4]***value[-4:]`.
2. `check_stale_envs(root: Path) -> list[str]`: Check for `.env` files in `CE - Agent Builder/`, `CE - Multi-Agent Orchestration/`, `CE - Evals/` under root. Return list of paths that exist (stale).
3. `group_keys_by_project() -> dict[str, list[KeyMeta]]`: Group KEY_REGISTRY entries by project using `required_by` field. Include a "Docker / Database" group for docker-category keys.
4. `run_check()`: Main function that:
   a. Calls `find_and_load_dotenv()` (catch EnvironmentError to report but not crash)
   b. Builds Rich table with columns: Key, Value, Status, Required
   c. Groups output by project
   d. Prints stale file warnings
   e. Prints summary: X/Y keys set, Z warnings

5. `if __name__ == "__main__": run_check()` at bottom.
