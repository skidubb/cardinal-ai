# Plan 05: Delete per-project .env files and run end-to-end verification

## Objective
Delete all per-project `.env` and `.env.example` files (replaced by root equivalents), then run end-to-end verification across all projects to confirm the consolidated environment works correctly from any CWD.

## must_haves
- `CE - Agent Builder/.env` deleted
- `CE - Multi-Agent Orchestration/.env` deleted
- `CE - Evals/.env` deleted
- `CE - Agent Builder/.env.example` deleted (replaced by root `.env.example`)
- `CE - Multi-Agent Orchestration/.env.example` deleted (replaced by root `.env.example`)
- `CE - Agent Builder/demo/.env.example` deleted (replaced by root `.env.example`)
- No `.env` file exists anywhere except repo root (excluding venv directories)
- Running `csuite` CLI from Agent Builder directory loads root `.env` and works
- Running a protocol from Orchestration directory loads root `.env` and works
- Removing ANTHROPIC_API_KEY from root `.env` causes immediate startup failure with clear error
- `python -m ce_shared.env_check` reports clean state (no stale files, all keys present)
- `docker compose config` still validates cleanly
