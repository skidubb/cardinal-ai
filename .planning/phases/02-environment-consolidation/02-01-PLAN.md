# Plan 01: Create ce-shared env module with loader, registry, and validation

## Objective
Add an `env` module to `ce-shared` that provides: (1) `find_and_load_dotenv()` ‚Äî walks up from CWD via `pathlib.Path.parents` to find the monorepo root `.env`, loads it with `python-dotenv` (override=False so shell exports take precedence), and runs validation; (2) `KEY_REGISTRY` ‚Äî a dict of `KeyMeta` dataclasses documenting every env var, its required_by projects, category, and whether it's required or optional; (3) `validate_env()` ‚Äî checks all registry keys against `os.environ`, raises `EnvironmentError` with actionable message for missing required keys, logs warnings for missing optional keys, returns the loaded `.env` path.

This is the foundation all subsequent plans depend on.

## must_haves
- `ce-shared/pyproject.toml` adds `python-dotenv>=1.0.0` and `rich>=13.0` as dependencies
- `ce-shared/src/ce_shared/env.py` exists with `find_and_load_dotenv()`, `KEY_REGISTRY`, `KeyMeta`, and `validate_env()`
- `find_and_load_dotenv()` traverses parents from CWD looking for `.env` with a monorepo sentinel (sibling `ce-shared/` directory)
- `find_and_load_dotenv()` calls `load_dotenv(dotenv_path, override=False)` then `validate_env()`
- Missing required key raises `EnvironmentError` with key name, purpose, and fix instructions
- Missing optional key emits `logging.warning()` (no crash)
- Shell-exported vars take precedence over `.env` file values (override=False)
- `find_and_load_dotenv()` returns the Path of the loaded `.env` file (or None if not found)
- `ce-shared/__init__.py` re-exports `find_and_load_dotenv`, `KEY_REGISTRY`, `KeyMeta`, `validate_env`
- Unit tests cover: parent traversal finds root .env, validation passes with all required keys, validation fails on missing required key, optional missing key warns but doesn't fail, override=False behavior

<task id="02-01-02">
Create `ce-shared/src/ce_shared/env.py` with:

1. `KeyMeta` dataclass: `name: str`, `required_by: list[str]`, `category: str`, `description: str`, `required: bool = True`
2. `KEY_REGISTRY: dict[str, KeyMeta]` containing ALL keys from the research inventory (Section 2). Use the standardized names from context decisions:
   - `LANGFUSE_BASE_URL` (not `LANGFUSE_HOST`)
   - `NOTION_API_KEY` (not `NOTION_TOKEN`)
   - Categories: "llm", "observability", "storage", "search", "integration", "config", "docker"
   - Required keys: `ANTHROPIC_API_KEY` only (the one universal requirement)
   - All others: `required=False` (optional with warning)
3. `_find_dotenv() -> Path | None`: Walk up from `Path.cwd()` via `.parents`, at each level check if `.env` file exists AND `ce-shared/` directory exists as sibling (monorepo sentinel). Return the `.env` Path or None.
4. `validate_env(project: str | None = None) -> list[str]`: Iterate KEY_REGISTRY. If `project` is specified, only check keys where project is in `required_by`. For each key: if required and missing from `os.environ`, collect into errors list. If optional and missing, emit `logging.warning()`. If errors list is non-empty, raise `EnvironmentError` with all missing keys, their descriptions, and "Add to .env at repo root or export in shell" instructions. Return list of warnings.
5. `find_and_load_dotenv(project: str | None = None) -> Path | None`: Call `_find_dotenv()`. If found, call `load_dotenv(str(path), override=False)`. Then call `validate_env(project)`. Return the path (or None). Log the loaded path at INFO level.
