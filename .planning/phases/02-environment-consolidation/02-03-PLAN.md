# Plan 03: Migrate all load_dotenv call sites and pydantic-settings to ce-shared loader

## Objective
Replace every `load_dotenv()` call and every pydantic-settings `env_file=".env"` across all 4 projects with `ce_shared.env.find_and_load_dotenv()`. Also migrate ce-db to construct DATABASE_URL from individual POSTGRES_* vars loaded by the shared loader. This ensures all projects load from the single root `.env` regardless of CWD.

## must_haves
- All 9 `load_dotenv()` call sites from research Section 3.1 replaced with `find_and_load_dotenv()`
- Both pydantic-settings `env_file=".env"` references removed (Agent Builder config.py, Evals config.py) — rely on env vars loaded into `os.environ`
- `find_and_load_dotenv()` is called BEFORE any `Settings()` instantiation or env var reads (addresses risks R-5, R-6, R-7)
- CE - Evals `pyproject.toml` adds `ce-shared @ file:../ce-shared` dependency
- ce-db `pyproject.toml` adds `ce-shared @ file:../ce-shared` dependency
- ce-db `engine.py` constructs `DATABASE_URL` from `POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_HOST`, `POSTGRES_DB`, `POSTGRES_PORT` — no hardcoded fallback URL
- Langfuse tracing module-level load happens at import time (preserving R-7 timing)
- All scripts in Agent Builder work when run from any CWD (addresses R-8)
